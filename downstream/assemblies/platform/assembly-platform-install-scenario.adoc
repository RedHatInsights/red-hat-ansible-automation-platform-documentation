= Installing {PlatformName}


:context: platform-install-scenario

[role="_abstract"]
{PlatformNameShort} is a modular platform and you can deploy {ControllerName} with other automation platform components, such as {HubName}. For more information about the components provided with {PlatformNameShort}, see _Red Hat Ansible Automation Platform platform components_ in the {PlatformName} Planning Guide.

[IMPORTANT]
====
With the {PlatformNameShort} installer, you can deploy *only one* {HubName} per inventory. You can use the {PlatformNameShort} installer for a standalone instance of {HubName} and run the installer any number of times with any number of different inventories to deploy multiple {HubName} nodes.
====

You can choose one of the supported installation scenarios for {PlatformName}. Edit the inventory file parameters to specify your installation scenario by using one of the following examples:

* xref:ref-standlone-platform-inventory_platform-install-scenario[Example inventory file for a database on the automation controller node or a non-installer managed database]
* xref:ref-standlone-platform-ext-database-inventory_platform-install-scenario[Example {PlatformName} inventory file with an external managed database]
* xref:ref-single-node-inventory_platform-install-scenario[Example {PlatformName} single node inventory file]
* xref:standalone-controller-ext-db_platform-install-scenario[Example inventory file for a standalone automation controller with an external managed database]
* xref:ref-standlone-hub-inventory_platform-install-scenario[Example standalone {HubName} inventory file]
* xref:ref-standlone-hub-ext-database-inventory_platform-install-scenario[Example standalone {HubName} inventory file]
* xref:ref-multi-node-cluster-inventory_platform-install-scenario[Example {PlatformName} multi-node inventory file]
